# 1. 設計原則

Surfin Batch Frameworkは、以下の原則に基づいて設計されています。

1. **JSR-352 の尊重**: ジョブ、ステップ、チャンク、タスクレットといった標準的な概念を継承し、予測可能性を担保。
2. **Go の並行処理の活用**: Goroutine と Channel を活用したパイプライン処理により、I/O 待ちと CPU 処理を並列化。
3. **アダプターによる疎結合**: データベース、ストレージ、オブザーバビリティなどの外部システム連携は、抽象インターフェース（Adapter）を介して行い、技術スタックへの依存を最小化。
4. **型安全性とジェネリクス**: Go のジェネリクスを活用し、コンパイル時の型チェックを徹底。
