# 0. プロジェクト概要

## 目的
Surfin バッチフレームワークは、エンタープライズレベルのバッチ処理における「規律」と、Go言語の「機動力」を融合させることを目的とします。

## 目指す姿
「外側は堅牢な JSR-352、中身は Go の Goroutine エンジン」。
このハイブリッドな構造により、大規模データ処理におけるパフォーマンスと、エンタープライズ環境で求められる信頼性を両立させます。

## 主要な設計原則
* **責務の分離 (Separation of Concerns)**: ジョブ定義、実行エンジン、ビジネスロジック、永続化層を明確に分離。
* **依存性逆転の原則 (DIP)**: コアロジックは抽象インターフェースに依存し、具体的なインフラ実装はアダプター層で提供。
* **拡張性 (Extensibility)**: ユーザー定義コンポーネント（Reader/Writer/Tasklet）の容易なプラグイン。
* **トランザクション境界の明確化**: ビジネスロジックからトランザクション管理を分離。
