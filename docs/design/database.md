# データベース設計

本ドキュメントは、Surfin バッチフレームワークにおけるデータベース関連の設計と戦略を統合したものです。

## 1. 設定戦略
データベース設定は `surfin.adapter.database` というネストされた構造で定義されます。
- **関心事の分離**: `core/config` は設定の読み込み、`adapter/database` は具体的な接続確立を担う。
- **柔軟性**: 環境変数による上書きをサポートし、柔軟なデプロイを可能にする。

## 2. Reader設計
### 2.1. SqlCursorReader
データベースカーソルを利用して1行ずつ効率的に読み込みます。再開性（Restartability）をサポートします。
- **目的**: データベースから大規模データを効率的に読み込むためのカーソルベース Reader。
- **設計**:
    - **ストリーミング**: `sql.Rows` を利用し、メモリに全データをロードしない。
    - **再開性**: `ExecutionContext` に読み込み位置（オフセットやID）を保持し、中断からの再開を可能にする。

## 3. Auto-Scan設計
`*sql.Rows` から Go 構造体への自動スキャン機能です。ORMの自動マッピング機能を活用し、ボイラープレートコードを削減します。
- **目的**: SQL クエリ結果から Go 構造体へのマッピングを自動化する。
- **設計**:
    - **リフレクション活用**: `sql.Rows` のカラム名と構造体のタグを照合し、値を自動セットする。
    - **ボイラープレート削減**: 手動での `rows.Scan` を不要にし、開発効率を向上させる。

## 4. SqlBulkWriter 設計
`SqlBulkWriter` は、大量データを効率的にデータベースに書き込むためのコンポーネントです。Spring Batchの `JdbcBatchItemWriter` の思想に基づき、バルク書き込みとトランザクション管理を統合します。

- **目的**: 大量データの高速なデータベース書き込み。
- **設計思想**:
    - **トランザクション参加**: フレームワークが提供するトランザクションに「参加」し、自身でトランザクションを開始・コミットしません。
    - **バルク処理**: `bulkSize` に基づいてアイテムをチャンク分割し、効率的に書き込みます。
- **構造体定義**:
    ```go
    type SqlBulkWriter[T any] struct {
        bulkSize        int      // 一度にコミットするアイテムの最大数
        name            string   // Writerの名前
        tableName       string   // 対象となるテーブル名
        conflictColumns []string // UPSERT時の競合カラム
        updateColumns   []string // UPSERT時に更新するカラム
    }
    ```
